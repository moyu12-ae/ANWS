# ADR-005: CLI 终端输出体验与品牌化表达

**状态**: Accepted
**日期**: 2026-03-14
**关联需求**: [REQ-003]
**关联版本**: .anws v4

---

## 背景

在 v3 之前，`anws` 的终端输出以“能说明结果”为主，但缺少产品级结构与辨识度。随着 `anws` 开始承担多工具初始化与升级入口，CLI 输出不再是附属文本，而是用户对产品的第一感知界面。

---

## 决策

将 CLI 输出体验视为正式架构对象，采用 **品牌头部 + 信息面板 + 状态树 + 下一步引导** 的结构化模型。

### 输出结构

1. **Header / Brand**
   - 使用 `anws` 自有 ASCII Logo 或统一标题头

2. **Summary Panel**
   - 展示目标工具、工作路径、输出路径、关键模式等摘要

3. **Progress Tree**
   - 展示步骤进度与结果

4. **Notice Panel**
   - 用于安全提示、兼容性提示、凭证目录提示

5. **Next Steps**
   - 用于引导用户进入 `/genesis`、`/upgrade` 或目标工具中的后续动作

---

## 选择理由

- 终端输出是产品体验的重要组成部分
- 结构化输出能显著降低理解成本
- 与优秀脚手架工具的用户预期一致
- 不需要图形界面，也能建立品牌识别

---

## 约束

1. 运行时零依赖
2. 必须跨平台可读
3. 颜色失效时仍需保留结构可读性
4. 输出要克制，不能为了好看牺牲信息密度

---

## 实施边界

- 该决策定义的是输出结构与体验原则
- 不要求 v4 立即实现最终视觉稿，但要求后续实现遵守该结构
- 如果某个目标工具环境不适合完整品牌输出，应保留最小兼容模式
